## 1. 프로젝트 개요

### 배경 및 문제 정의

| 구분 | 내용 |
|---|---|
| **AS-IS** | 패션 상품 이미지는 MD가 수작업으로 태깅하거나 파일명 기반으로 관리되고 있음<br>  - 이미지 검색이 텍스트 키워드에 의존<br>- 스타일 / 분위기 기반 검색이 불가능<br>- 유사 상품 탐색 및 레퍼런스 분석이 어려움<br>- 이미지 자산이 데이터화되지 못하고 단순 파일 형태로 관리|
| **TO-BE** | 이미지를 AI가 이해 가능한 데이터로 변환하고, 자연어 및 이미지 기반으로 유사 상품을 검색할 수 있는 시스템 구축 |

### 목표

| 구분 | 진행기간 | 내용 |
|---|---|---|
| 1차 목표 | 2.2 ~ 2.9 | 이미지에 AI 기반 자동 태깅 적용, Pinecone 벡터 DB 저장 |
| 2차 목표 | 2.9 ~ 2.19 | 자연어 텍스트 / 이미지 업로드 기반 유사 상품 검색 구현 |
| 3차 목표 | 2.19 ~ 2.27 | 업로드 및 검색 정확도 개선 |

## 3. 시스템 아키텍처

### 전체 흐름

```

     ↓
Gemini Vision 태깅
     ↓

     ↓
Pinecone Vector DB 저장
     ↓
텍스트 / 이미지 검색
```

### 구성 요소
```
upload.py

search.py
 ├ 텍스트 검색
 ├ 이미지 검색
 └ 웹 유사도 검색
```

## 4. 주요 기능

### 4-1. 이미지 업로드 및 자동 태깅 (`upload.py`)

#### Step 1. 브랜드 자동 분류
파일명 패턴을 파싱해 브랜드를 자동 추출한다 (`brand_data.py`).

#### Step 2. 이미지 분위기(Vibe) 추출
Gemini Vision이 이미지를 보고 2~3문장의 한글 감성 묘사를 생성한다.

```python
# 예시 출력
"차분하고 고급스러운 무드의 클래식한 자켓이다. 
 블랙 컬러와 정제된 실루엣이 포멀한 느낌을 주며, 
 하객룩이나 비즈니스 캐주얼로 활용하기 좋다."
```

이 텍스트는 **검색 쿼리와의 의미론적 유사도**를 높이는 핵심 역할을 한다.

#### Step 3. 구조화 태그 추출 (JSON)
MD 20년 경력 페르소나를 부여한 프롬프트로 구조화 태그를 생성한다.

```json
{
  "cat": "자켓",
  "col": ["블랙"],
  "mat": "트위드",
  "sty": "클래식",
  "fit": "슬림핏",
  "gen": "여성",
  "age": "3040",
  "det": ["골드 버튼", "패드 숄더"]
}
```

#### Step 4. 통합 임베딩
vibe 텍스트 + 메타데이터를 하나의 문자열로 결합 후 1408차원 벡터로 변환한다.

```
"{vibe 2~3문장} {brand} {cat} {colors} {sty} {mat} {fit} {det}"
        ↓
  gemini-embedding-001 (1408d)
        ↓
  Pinecone upsert
```

#### Step 5. Pinecone 저장
벡터와 메타데이터를 함께 저장해 검색 시 원본 이미지 링크 및 태그를 즉시 반환할 수 있게 한다.

---

### 4-2. 텍스트 기반 벡터 검색 (`search.py` 탭1)

자연어 입력 → Gemini 구조화 → 동일한 임베딩 형식의 `query_text` 생성 → Pinecone 검색의 3단계 파이프라인이다.

```
"결혼식 하객룩 블랙 자켓 추천해줘"
        ↓  [Gemini 구조화]
query_text: "클래식하고 단정한 포멀 분위기의 고급스러운 룩이다.
             하객룩으로 잘 어울리는 세련된 스타일이다.
             자켓 블랙 트위드 클래식 포멀 하객룩 결혼식"
        ↓  [1408차원 임베딩]
        ↓  [Pinecone ANN 검색, top_k=50]
        ↓  [post-filter: 성별/카테고리/연령/유사도]
     최종 결과 (최대 12건, score ≥ 0.5)
```

**저장 시와 검색 시의 텍스트 형식을 일치**시키는 것이 검색 정확도의 핵심이다.

---

### 4-3. 이미지 기반 벡터 검색 (`search.py` 탭2)

업로드된 이미지에서 upload.py와 동일한 방식으로 vibe + 태그를 추출하고, 해당 텍스트를 임베딩해 Pinecone에서 유사 아이템을 검색한다.

```
[업로드 이미지]
      ↓
  vibe 추출 + 태그 JSON 추출  (upload.py와 완전 동일한 로직)
      ↓
  combined_text = vibe + metadata_text
      ↓
  1408차원 임베딩 → Pinecone 검색
      ↓
  post-filter (이미지에서 추출한 gen/age/cat 활용)
```

---

### 4-4. 웹 유사도 검색 (`search.py` 탭3)

Google Cloud Vision `WEB_DETECTION` API를 활용한다. Pinecone DB와 무관하게 인터넷 전체에서 시각적으로 유사한 이미지와 출처 페이지를 찾는다. 경쟁사 유사 상품 조사, 트렌드 분석 등 다른 목적으로 활용 가능하다.

### 4-5. Google Drive 이미지 로딩 (서버사이드 bytes 방식)

#### 문제
`st.image(URL)`은 브라우저가 직접 Google Drive URL을 요청한다. Drive CDN은 인증/CORS/캐시 문제로 간헐적으로 403을 반환해 이미지가 깨진다.

#### 해결
Python 서버가 bytes를 직접 받아 `st.image(bytes)`로 렌더링한다.

```
브라우저 → Streamlit 서버 → Google Drive
                ↑
        서버가 bytes 수신
        base64 인코딩 후 HTML에 내장
                ↓
        브라우저는 인증 불필요
```

3단계 fallback으로 안정성을 확보한다.

```python
1순위: Bearer 토큰 → Drive API 
2순위: lh3.googleusercontent.com CDN
3순위: drive.google.com/thumbnail 
```

`@st.cache_data(ttl=3600)`으로 세션 내 중복 호출을 방지하고, `ThreadPoolExecutor(max_workers=8)`로 결과 렌더링 전 병렬 prefetch를 수행한다.

## 5. 데이터 파이프라인 상세

### 업로드 파이프라인 (`upload.py`)

```
이미지 파일 선택 (최대 10장)
    │
    ├─ 브랜드 추출 (파일명 파싱)
    ├─ UUID 파일명 생성
    ├─ Google Drive 업로드 → webViewLink 획득
    │
    ├─ [Gemini Vision] vibe 텍스트 생성
    ├─ [Gemini Vision] 태그 JSON 생성
    │
    ├─ combined_text 조합
    ├─ [gemini-embedding-001] 1408차원 벡터 생성
    │
    └─ Pinecone upsert (벡터 + 메타데이터)
```

API Rate Limit 대응으로 파일 간 2초 sleep을 적용했다 (무료 티어 기준 분당 15회 제한).

### 검색 파이프라인 (`search.py`)

```
사용자 입력 (텍스트 또는 이미지)
    │
    ├─ [Gemini] 쿼리 구조화 (gen, age, cat, occasion, query_text 추출)
    │
    ├─ [gemini-embedding-001] query_text → 1408차원 벡터
    │
    ├─ Pinecone query (top_k=50, include_metadata=True)
    │   └─ detail_json 파싱 → gen, age 추출
    │
    ├─ post-filter
    │   ├─ 성별 하드 필터 (완전 제거)
    │   ├─ 카테고리 소프트 필터 (×1.5 / ×0.35)
    │   ├─ 연령대 소프트 필터 (×1.1 / ×0.7)
    │   └─ 유사도 임계값 (score < 0.5 제거)
    │
    └─ 상위 12건 렌더링
```

## 6. 향후 고도화 방안
### 1️⃣ 멀티모달 검색 고도화

현재는 텍스트 기반 임베딩 중심 구조로 동작하고 있다.  
향후 CLIP 기반 이미지 임베딩을 추가하여 다음과 같은 기능을 구현할 수 있다.

- 이미지 + 텍스트 멀티모달 검색
- 더 정밀한 시각적 유사도 기반 검색

---

### 2️⃣ 상품 데이터 연동

향후 ERP 및 상품 DB와 연동하여 다음 데이터를 함께 활용할 수 있다.

- 상품 코드
- 가격
- 시즌
- 판매 데이터
